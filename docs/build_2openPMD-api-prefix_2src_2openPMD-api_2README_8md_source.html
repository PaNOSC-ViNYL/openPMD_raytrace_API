25 openPMD is an open meta-data schema that provides meaning and self-description for data sets in science and engineering.
26 See [the openPMD standard](https://github.com/openPMD/openPMD-standard) for details of this schema.
27 
28 This library provides a reference API for openPMD data handling.
29 Since openPMD is a schema (or markup) on top of portable, hierarchical file formats, this library implements various backends such as HDF5, ADIOS1, ADIOS2 and JSON.
30 Writing & reading through those backends and their associated files is supported for serial and [MPI-parallel](https://www.mpi-forum.org/docs/) workflows.

33 
34 ### C++
35 
36 [![C++11][api-cpp]](https://isocpp.org/) ![C++11 API: Alpha][dev-alpha]
37 
38 [api-cpp]: https://img.shields.io/badge/language-C%2B%2B11-yellowgreen "C++11 API"
39 [dev-alpha]: https://img.shields.io/badge/phase-alpha-yellowgreen "Status: Alpha"
40 
41 ```cpp
42 #include <openPMD/openPMD.hpp>
43 #include <iostream>
44 
45 // ...
46 
47 auto s = openPMD::Series("samples/git-sample/data%T.h5", openPMD::Access::READ_ONLY);
48 
49 for( auto const& i : s.iterations ) {
50  std::cout << "Iteration: " << i.first << "\n";
51 
52  for( auto const& m : i.second.meshes ) {
53  std::cout << " Mesh '" << m.first << "' attributes:\n";
54  for( auto const& val : m.second.attributes() )
55  std::cout << " " << val << '\n';
56  }
57 
58  for( auto const& p : i.second.particles ) {
59  std::cout << " Particle species '" << p.first << "' attributes:\n";
60  for( auto const& val : p.second.attributes() )
61  std::cout << " " << val << '\n';
62  }
63 }
64 ```

66 ### Python
67 
68 [![Python3][api-py3]](https://www.python.org/) ![Python3 API: Alpha][dev-alpha]

73 ```py
74 import openpmd_api as io
75 
76 # ...
77 
78 series = io.Series("samples/git-sample/data%T.h5", io.Access.read_only)
79 
80 for k_i, i in series.iterations.items():
81  print("Iteration: {0}".format(k_i))
82 
83  for k_m, m in i.meshes.items():
84  print(" Mesh '{0}' attributes:".format(k_m))
85  for a in m.attributes:
86  print(" {0}".format(a))
87 
88  for k_p, p in i.particles.items():
89  print(" Particle species '{0}' attributes:".format(k_p))
90  for a in p.attributes:
91  print(" {0}".format(a))
92 ```

109 * [NLohmann-JSON](https://github.com/nlohmann/json) 3.7.0+ ([MIT](https://github.com/nlohmann/json/blob/develop/LICENSE.MIT))
110 
111 I/O backends:
112 * [JSON](https://en.wikipedia.org/wiki/JSON)

115 * [ADIOS2](https://github.com/ornladios/ADIOS2) 2.6.0+ (optional)

118 * MPI 2.1+, e.g. OpenMPI 1.6.5+ or MPICH2

121 * Python:

210 openPMD-api can also be built and installed from source using [CMake](https://cmake.org/):
211 
212 ```bash
213 git clone https://github.com/openPMD/openPMD-api.git
214 
215 mkdir openPMD-api-build
216 cd openPMD-api-build
217 
218 # optional: for full tests, with unzip
219 ../openPMD-api/share/openPMD/download_samples.sh
220 
221 # for own install prefix append:
222 # -DCMAKE_INSTALL_PREFIX=$HOME/somepath
223 # for options append:
224 # -DopenPMD_USE_...=...
225 # e.g. for python support add:
226 # -DopenPMD_USE_PYTHON=ON -DPYTHON_EXECUTABLE=$(which python3)
227 cmake ../openPMD-api
228 
229 cmake --build .
230 
231 # optional
232 ctest
233 
234 # sudo might be required for system paths
235 cmake --build . --target install
236 ```
237 
238 The following options can be added to the `cmake` call to control features.
239 CMake controls options with prefixed `-D`, e.g. `-DopenPMD_USE_MPI=OFF`:
240 
241 | CMake Option | Values | Description |
242 |------------------------------|------------------|------------------------------------------------------------------------------|
243 | `openPMD_USE_MPI` | **AUTO**/ON/OFF | Parallel, Multi-Node I/O for clusters |
244 | `openPMD_USE_HDF5` | **AUTO**/ON/OFF | HDF5 backend (`.h5` files) |
245 | `openPMD_USE_ADIOS1` | **AUTO**/ON/OFF | ADIOS1 backend (`.bp` files up to version BP3) |
246 | `openPMD_USE_ADIOS2` | **AUTO**/ON/OFF | ADIOS2 backend (`.bp` files in BP3, BP4 or higher) |
247 | `openPMD_USE_PYTHON` | **AUTO**/ON/OFF | Enable Python bindings |
248 | `openPMD_USE_INVASIVE_TESTS` | ON/**OFF** | Enable unit tests that modify source code <sup>1</sup> |
249 | `openPMD_USE_VERIFY` | **ON**/OFF | Enable internal VERIFY (assert) macro independent of build type <sup>2</sup> |
250 | `openPMD_INSTALL` | **ON**/OFF | Add installation targets |
251 | `PYTHON_EXECUTABLE` | (first found) | Path to Python executable |
252 
253 <sup>1</sup> *e.g. changes C++ visibility keywords, breaks MSVC*
254 <sup>2</sup> *this includes most pre-/post-condition checks, disabling without specific cause is highly discouraged*
255 
256 Additionally, the following libraries are shipped internally.
257 The following options allow to switch to external installs:
258 
259 | CMake Option | Values | Library | Version |
260 |---------------------------------|------------|---------------|---------|
261 | `openPMD_USE_INTERNAL_VARIANT` | **ON**/OFF | MPark.Variant | 1.4.0+ |
262 | `openPMD_USE_INTERNAL_CATCH` | **ON**/OFF | Catch2 | 2.6.1+ |
263 | `openPMD_USE_INTERNAL_PYBIND11` | **ON**/OFF | pybind11 | 2.3.0+ |
264 | `openPMD_USE_INTERNAL_JSON` | **ON**/OFF | NLohmann-JSON | 3.7.0+ |
265 
266 By default, this will build as a shared library (`libopenPMD.[so|dylib|dll]`) and installs also its headers.
267 In order to build a static library, append `-DBUILD_SHARED_LIBS=OFF` to the `cmake` command.
268 You can only build a static or a shared library at a time.
269 
270 By default, the `Release` version is built.
271 In order to build with debug symbols, pass `-DCMAKE_BUILD_TYPE=Debug` to your `cmake` command.
272 
273 By default, tests, examples and command line tools are built.
274 In order to skip building those, pass `-DBUILD_TESTING=OFF`, `-DBUILD_EXAMPLES=OFF`, or `-DBUILD_CLI_TOOLS=OFF` to your `cmake` command.

276 ## Linking to your project
277 
278 The install will contain header files and libraries in the path set with `-DCMAKE_INSTALL_PREFIX`.
279 
280 ### CMake
281 
282 If your project is using CMake for its build, one can conveniently use our provided `openPMDConfig.cmake` package which is installed alongside the library.
283 
284 First set the following environment hint if openPMD-api was *not* installed in a system path:
285 
286 ```bash
287 # optional: only needed if installed outside of system paths
288 export CMAKE_PREFIX_PATH=$HOME/somepath:$CMAKE_PREFIX_PATH
289 ```
290 
291 Use the following lines in your project's `CMakeLists.txt`:
292 ```cmake
293 # supports: COMPONENTS MPI NOMPI HDF5 ADIOS1 ADIOS2
294 find_package(openPMD 0.9.0 CONFIG)
295 
296 if(openPMD_FOUND)
297  target_link_libraries(YourTarget PRIVATE openPMD::openPMD)
298 endif()
299 ```
300 
301 *Alternatively*, add the openPMD-api repository source directly to your project and use it via:
302 ```cmake
303 add_subdirectory("path/to/source/of/openPMD-api")
304 
305 target_link_libraries(YourTarget PRIVATE openPMD::openPMD)
306 ```
307 
308 For development workflows, you can even automatically download and build openPMD-api from within a depending CMake project.
309 Just replace the `add_subdirectory` call with:
310 ```cmake
311 include(FetchContent)
312 set(CMAKE_POLICY_DEFAULT_CMP0077 NEW)
313 set(BUILD_CLI_TOOLS OFF)
314 set(BUILD_EXAMPLES OFF)
315 set(BUILD_TESTING OFF)
316 set(openPMD_INSTALL ${BUILD_SHARED_LIBS}) # e.g. only install if used as shared library
317 set(openPMD_USE_PYTHON OFF)
318 FetchContent_Declare(openPMD
319  GIT_REPOSITORY "https://github.com/openPMD/openPMD-api.git"
320  GIT_TAG "dev")
321 FetchContent_MakeAvailable(openPMD)
322 ```

324 ### Manually
325 
326 If your (Linux/OSX) project is build by calling the compiler directly or uses a manually written `Makefile`, consider using our `openPMD.pc` helper file for `pkg-config` which are installed alongside the library.
327 
328 First set the following environment hint if openPMD-api was *not* installed in a system path:
329 
330 ```bash
331 # optional: only needed if installed outside of system paths
332 export PKG_CONFIG_PATH=$HOME/somepath/lib/pkgconfig:$PKG_CONFIG_PATH
333 ```
334 
335 Additional linker and compiler flags for your project are available via:
336 ```bash
337 # switch to check if openPMD-api was build as static library
338 # (via BUILD_SHARED_LIBS=OFF) or as shared library (default)
339 if [ "$(pkg-config --variable=static openPMD)" == "true" ]
340 then
341  pkg-config --libs --static openPMD
342  # -L/usr/local/lib -L/usr/lib/x86_64-linux-gnu/openmpi/lib -lopenPMD -pthread /usr/lib/libmpi.so -pthread /usr/lib/x86_64-linux-gnu/openmpi/lib/libmpi_cxx.so /usr/lib/libmpi.so /usr/lib/x86_64-linux-gnu/hdf5/openmpi/libhdf5.so /usr/lib/x86_64-linux-gnu/libsz.so /usr/lib/x86_64-linux-gnu/libz.so /usr/lib/x86_64-linux-gnu/libdl.so /usr/lib/x86_64-linux-gnu/libm.so -pthread /usr/lib/libmpi.so -pthread /usr/lib/x86_64-linux-gnu/openmpi/lib/libmpi_cxx.so /usr/lib/libmpi.so
343 else
344  pkg-config --libs openPMD
345  # -L${HOME}/somepath/lib -lopenPMD
346 fi
347 
348 pkg-config --cflags openPMD
349 # -I${HOME}/somepath/include
350 ```
