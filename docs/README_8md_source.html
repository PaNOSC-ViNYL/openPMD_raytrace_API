1 # c++ API for the openPMD ray trace extension
2 
3 This library provides an API in C++ and Python to read and write openPMD files following the raytrace extension.

6 ## Quantities
7 According to the extension, here's the list of variables stored in the file and the corresponding units.

10 | Variable | Comment | Units |
11 | --------- | ---------------- | ------------------------ |
12 | x,y,z | position in | [cm] |
13 | dx,dy,dz | direction | (normalized velocity) |
14 | sx,sy,sz | polarization of non-photons | |
15 | sPolAx,sPolAy,sPolAz | s-polarization amplitude of photons | |
16 | pPolAx,pPolAy,pPolAz | p-polarization amplitude of photons | |
17 | sPolx,sPoly,sPolz | s-polarization amplitude of photons | |
18 | pPolx,pPoly,pPolz | p-polarization amplitude of photons | |
19 | time | ray time w.r.t. ray generation | [ms] |
20 | wavelength | | [Ang] |
21 | weight | weight | |
22 | | | |

79 ## Unit conversion
80 
81 The units of the quantities stored in the openPMD file are pre-defined by the extension and not customizable by the user.
82 
83 For a more user-friendly user interface, the user should create it's own version of the Ray object (either in C++ or in Python), inheriting from the original Ray object.
84 The conversion should be done in the overloaded methods.
85 This should highly simplify the user code.
86 
87 Otherwise, the user can also use the bare Ray class with the "scale" optional argument for the different methods to scale the units into the pre-defined ones.
